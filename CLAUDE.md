# 未完成之书

The Book of Unfinished Things

## 项目定位

这是一个融合塔罗、雷诺曼、开放式提问、黑暗油画视觉、死亡文化阅读与私人记录的艺术化互动网站。

**它不是：**

- 普通塔罗网站
- 算命工具
- 恐怖游戏
- SaaS 控制台
- 小说写作工具
- 世界观设定管理器
- 死亡猎奇百科

## 核心理念

有些问题没有答案。
它们只是在暗处，慢慢显出形状。

核心动作是「显影」，不是「预言」。

网站不负责替用户决定，也不声称准确预测未来。
它通过图像、象征、文字和时间，让一些原本模糊的事物暂时变得可见。

## 长期开发规则

### 技术

- 所有代码使用 TypeScript。
- 默认使用 React Server Components。
- 只有确实需要浏览器交互时才使用 `"use client"`。
- 不要把所有组件都做成 client components。
- 不要擅自安装依赖。
- 不要擅自引入 UI 组件库。
- 不要使用 shadcn/ui 的默认审美。
- 不要引入 Radix UI，除非后续明确要求。
- 不要引入 Tiptap，除非后续明确要求。
- 不要引入动画库，除非后续明确要求。
- 不要添加数据库、账号系统、支付系统或 AI 接口，除非后续明确要求。

### 内容

- 不要生成大量未经审核的死亡文化内容。
- 不要伪造参考资料。
- 不要编造文化习俗、民族资料、学者、机构、书籍、统计数字。
- 不要把任何国家或文化描述成单一、固定、统一的死亡实践。
- 不要浪漫化自杀、自残或危险行为。
- 不要用恐吓式占卜文案。
- 占卜内容只能用于娱乐、创作和自我反思。
- 不要把「死神」牌解释为用户或他人的生理死亡。

### 流程

- 每个阶段结束必须运行 `npm run lint` 和 `npm run build`。
- 每次修改后必须汇报修改文件、检查结果和下一步建议。

## 视觉禁止项

- 禁止紫色星空塔罗模板。
- 禁止水晶球、廉价星星、魔法阵堆叠。
- 禁止廉价骷髅、血迹飞溅、恐怖游戏风格。
- 禁止大面积霓虹渐变。
- 禁止普通 SaaS 仪表盘风格。
- 禁止过度圆角卡片集合。
- 禁止把页面做成模板站。
- 禁止自动播放音乐。

## 视觉方向

黑暗油画 × 超现实静物 × 当代编辑设计

- 油画负责情绪。
- 界面负责秩序。
- 以黑暗静物油画为主。
- 以超现实遗物作为象征变化。
- 以少量腐朽浪漫主义场景作为扩展。
- 页面应像安静的深夜画廊，而不是恐怖场景。
- 用户第一眼应感到克制、文学、安静、神秘但不廉价。

## 主要栏目

| 栏目 | 说明 |
|------|------|
| 未明之问 | 开放式提问与塔罗、雷诺曼抽牌 |
| 显影 | 抽牌过程与结果体验 |
| 告别志 | 死亡、仪式、纪念与失去的文化阅读 |
| 旧问录 | 用户过去的问题、抽牌记录与后记 |
| 物的余生 | 连接卡牌、油画、死亡文化与个人经验的意象索引 |
| 书后 | 项目说明、内容边界与使用声明 |

## 关键体验原则

- 先观察，再解释。
- 用户先看油画，再阅读牌义。
- 每张牌同时有传统身份和《未完成之书》的艺术身份。
- 抽牌结果应是一页可以未来重读的记录，而不是一次性答案。
- 用户可以自由命名主题。
- 用户可以不输入问题。
- 旧问录应允许用户补写「后来」和「如今」。
- 告别志以问题和物件组织内容，不以猎奇国家清单组织内容。
